### “我要重构”

## *德霖进行了第二次重构*

这次重构我的想法是创建一个好看，简洁，功能强的界面，本质其实区别不大。

**当然最主要的是现实一种AI开发的方式**

第一步当然是通过画图理解并拆分自己的需求，这和通常软件开发逻辑应该相似，没有明确的目标是写不了代码的

第二步是写提示词，这是通过AI写代码很重要的一步，如果想生成一个复杂的而且还符合个人意愿的软件，需要非常全面的提示词。例如，这个主应用的提示词是


>写一个C#语言程序，在没有外置存储器的时候，上大半区域空白，中心显示请插入存储器。下小半区域显示选择存储器后查看。在插入外置存储器后，在上半区域显示一个小长方形区域，包含盘符，名字，图形化显示的盘容量大小和已使用大小。选择存储器后，存储器边框变绿，下半区域显示，分为五拦，第一栏为拷贝数据时间，第二栏为拷贝文件类型，可选照片，视频。第三栏为位置，可选拷贝位置和备份位置，备份可不选，第四栏为增加前缀和增加时间戳，前缀必加，第五栏为开始按钮。在选定好后，点击开始按钮，从所选存储器中，按所选时间把指定时间的所有指定类型文件拷贝到指定位置，对文件进行分类，把照片放到照片命名的文件夹中，视频放到视频命名的文件夹中，两个文件夹放到以前缀命名的大文件夹中，没有选择的文件类型不产生文件夹。分类好后，将文件夹中文件按前缀_时间戳_原文件名的方式重命名，时间戳的精确度到分钟，如果有备份路径，把拷出总文件夹复制一份到备份位置。这些过程都进行可视化放到原来显示存储器容量大小的位置，全部完成后，在小长方形后面区域显示对号，如果可以就实现多进程，同时拷贝多个存储器<
